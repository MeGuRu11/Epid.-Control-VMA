# Контекст проекта Epid Control

## 1. Что это за проект

`Epid Control` — Windows-first настольное приложение на `Python + PySide6 + SQLAlchemy + SQLite` для эпидемиологического контроля, ведения пациентов и госпитализаций, лабораторных и санитарных проб, аналитики, отчётов, `Form100 V2`, обмена данными и административных операций.

Проект развивается по модели `Clean Architecture / DDD`:

- `app/ui` — интерфейс и навигация;
- `app/application` — сервисы приложения, use-case и права;
- `app/domain` — правила предметной области;
- `app/infrastructure` — БД, миграции, отчёты, импорт/экспорт, файловые операции.

3. При необходимости отдельно привести исторические audit-отчёты к единому редакционному оформлению без изменения сути.

## 5. Ключевые правила проекта

- UI не импортирует `infrastructure` и не работает напрямую с SQLAlchemy.
- Каждая мутирующая операция проверяет права и знает `actor_id`.
- Все операции с БД идут через `session_scope()` или явно внедрённую session factory.
- DML SQL не строится через string interpolation.
- Персональные данные пациентов не логируются в открытом виде.
- После любого изменения кода обязательно запускаются `ruff`, `mypy`, `pytest`, `compileall`.

## 6. Карта основных разделов интерфейса

Разделы главного окна:

- `Главная`
- `ЭМЗ`
- `Поиск и ЭМК`
- `Лаборатория`
- `Санитария`
- `Аналитика`
- `Импорт/Экспорт`
- `Справочники`
- `Администрирование`

Сквозной рабочий элемент интерфейса — контекстная панель пациента. Через неё пользователь закрепляет пациента и госпитализацию, а затем переходит в связанные разделы без повторного поиска.

## 7. База данных и миграции

- Основная БД — локальная `SQLite`.
- Миграции — `Alembic`.
- Для локальной проверки схемы используется writable-каталог `tmp_run/epid-data`.
- Полнотекстовые FTS-объекты обслуживаются отдельно и не должны ломать обычный `alembic check`.

Полезные команды:

```powershell
python -m alembic upgrade head
python -m alembic current
python -m alembic heads
python -m alembic check
```
