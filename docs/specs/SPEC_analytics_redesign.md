# SPEC: Analytics Redesign — Regression Scenarios

**Дата:** 2026-05-09
**Назначение:** список ручных и автоматизированных сценариев, которые ОБЯЗАТЕЛЬНО должны работать в v2 после редизайна. Если хотя бы один не пройден — v2 не выкатывается в default.

---

## Принцип

Любая функциональность, доступная пользователю в v1, должна быть доступна в v2. Возможные изменения:
- расположение элемента (например, история отчётов перенесена во вкладку «Отчёты»);
- количество кликов (может сократиться, не должно увеличиться);
- визуальный стиль (полностью обновляется).

Запрещено:
- Удаление функции без явного согласования.
- Увеличение количества кликов до результата.
- Изменение результата (числа, выгружаемые файлы, SHA256 и т.п.).

## Раздел 1 — Фильтрация и базовый поиск

### 1.1 Период
- [ ] Quick period «Месяц» выставляет date_from = 1-е число текущего месяца, date_to = сегодня.
- [ ] Quick period «Неделя» выставляет date_from = 7 дней назад, date_to = сегодня.
- [ ] Quick period «Квартал» выставляет date_from = начало квартала, date_to = сегодня.
- [ ] Изменение date_from/date_to вручную сразу обновляет KPI и графики.
- [ ] При date_from > date_to показывается inline-предупреждение, поиск блокируется.

### 1.2 Расширенные фильтры
- [ ] Фильтр по отделению — single-select с поиском.
- [ ] Фильтр по ICD-10 — combo с динамической подгрузкой.
- [ ] Фильтр по микроорганизму — combo с поиском.
- [ ] Фильтр по антибиотику — combo с поиском.
- [ ] Фильтр по типу материала — combo.
- [ ] Фильтр по категории пациента — single-select.
- [ ] Фильтр «рост» (Любой/Только положительные/Только отрицательные).
- [ ] Сброс одного фильтра — кнопка «×» внутри chip.
- [ ] Сброс всех фильтров — кнопка «Сбросить» в шапке.
- [ ] Применение фильтра обновляет ВСЕ вкладки (Обзор, Микро, ИСМП).

### 1.3 Сохранённые фильтры
- [ ] Сохранение текущего набора фильтров под именем.
- [ ] Применение сохранённого фильтра — все поля заполняются.
- [ ] Удаление сохранённого фильтра.
- [ ] Список сохранённых фильтров отсортирован по дате создания (новые сверху).
- [ ] Имя фильтра уникально для пользователя.

## Раздел 2 — Аналитика и KPI

### 2.1 Сводка (Overview)
- [ ] KPI «Госпитализаций» = `ismp_metrics.total_cases`.
- [ ] KPI «Случаев ИСМП» = `ismp_metrics.ismp_cases`.
- [ ] KPI «Положительных» = `aggregates.positives` + доля `positive_share`.
- [ ] KPI «Превалентность» = `ismp_metrics.prevalence`.
- [ ] Trend-индикатор показывает % изменения относительно `compare_periods` (период такой же длительности назад).
- [ ] Trend ▲ зелёный для роста положительной метрики (например, число обследований).
- [ ] Trend ▼ красный для роста негативной метрики (число ИСМП, превалентность).
- [ ] При нулевом изменении показывается нейтральный «—».
- [ ] При отсутствии данных за прошлый период trend скрыт, не показывает «∞» или «N/A».

### 2.2 Trend chart
- [ ] Группировка «Авто» выбирается по длительности периода: до 35 дней — Дни, до 180 — Недели, дальше — Месяцы.
- [ ] Группировка «Дни» / «Недели» / «Месяцы» — корректное агрегирование.
- [ ] Сравнительная линия «прошлый период» (пунктир) показывается всегда, если данных хватает.
- [ ] Tooltip при наведении показывает дату и точное значение.
- [ ] Пустые дни без данных рисуются как 0, не как gap.

### 2.3 Топ микроорганизмов
- [ ] Bar chart показывает топ-10 микроорганизмов по количеству выделений.
- [ ] Каждый бар содержит код + название (например, «ECO — E. coli»).
- [ ] Клик по бару фильтрует все вкладки по выбранному микро.
- [ ] Если выделений < 10, показывается столько, сколько есть.

### 2.4 ИСМП
- [ ] 5 KPI: Случаев, Госпитализаций с ИСМП, Инцидентность ‰, Плотность ‰койко-дн, Превалентность %.
- [ ] Donut: распределение по 7 типам (ВАП / КА-ИК / КА-ИМП / ИОХВ / ПАП / БАК / СЕПСИС).
- [ ] Donut: легенда с количеством и процентом.
- [ ] Bar по отделениям: топ-10 по числу ИСМП.
- [ ] Все числа совпадают с теми, что считает `AnalyticsService.get_ismp_metrics()`.

### 2.5 Heatmap «Отделения × Микроорганизмы» (новая фича)
- [ ] Строки — топ-10 отделений по числу проб.
- [ ] Колонки — топ-10 микроорганизмов по числу выделений.
- [ ] Цветовая шкала: зелёный → жёлтый → красный по доле положительных в ячейке.
- [ ] Tooltip показывает количество и долю.
- [ ] Клик по ячейке фильтрует поиск по этому отделению + микро.

### 2.6 Resistance pattern (новая фича)
- [ ] Грид: топ-10 микроорганизмов × ключевые антибиотики (CIP, AMC, GEN, MEM, COL и т.п.).
- [ ] Цветовое кодирование по `lab_abx_susceptibility.ris`:
  - 🟢 «S» (sensitive) ≥ 80% — зелёный.
  - 🟡 20–80% — жёлтый.
  - 🔴 «R» (resistant) ≥ 50% или «I» доминирует — красный.
- [ ] При недостатке данных в ячейке (< 5 проб) показывается серый «—».
- [ ] Tooltip показывает абсолютное число (R/I/S) и общее.

## Раздел 3 — Поиск и результаты

### 3.1 Запуск поиска
- [ ] Кнопка «Найти» запускает `analytics_service.search_samples()` с текущими фильтрами.
- [ ] Во время запроса отображается loading-индикатор (skeleton или spinner).
- [ ] При ошибке показывается inline-плашка, не модальный диалог.

### 3.2 Таблица результатов
- [ ] Колонки: ID, Лаб. №, ФИО, Категория, Дата, Отделение, Материал, Микро, Антибиотик.
- [ ] Сортировка по любой колонке.
- [ ] Клик по строке открывает превью пробы (модальное окно или панель справа).
- [ ] При >1000 результатах используется виртуализация или пагинация.
- [ ] Color-coded badge: положительные пробы — мягкая красная подсветка, отрицательные — нейтральная.

### 3.3 Quick filter chips (новая фича)
- [ ] Chip «Только положительные» — клик добавляет growth_flag=True в фильтр.
- [ ] Chip «Только Грам−» / «Только Грам+» — фильтрует по taxon_group микроорганизма.
- [ ] Chip «Только из крови» / «Только из ран» — фильтрует по material_type.
- [ ] Активный chip визуально выделен (бирюзовый бордер).
- [ ] Повторный клик снимает фильтр.

## Раздел 4 — Экспорт

### 4.1 Excel
- [ ] Кнопка «Экспорт XLSX» — диалог сохранения с дефолтным именем (например, `analytics_2026-05-09.xlsx`).
- [ ] Файл содержит листы: Сводка, Фильтры, Данные.
- [ ] Числа — реальные числовые ячейки с правильным форматированием (даты — datetime, проценты — `0.0%`).
- [ ] При уже существующем файле — подтверждение перезаписи.
- [ ] После экспорта запись добавляется в историю (`report_run`).
- [ ] SHA256 экспорта сохраняется и проверяется.

### 4.2 PDF
- [ ] Кнопка «Экспорт PDF» — диалог сохранения.
- [ ] PDF содержит шапку, фильтры, summary, основные графики и таблицы.
- [ ] Данные совпадают с XLSX.
- [ ] Запись в `report_run` создана.

### 4.3 Quick-export всей вкладки (новая фича)
- [ ] Кнопка «Скачать отчёт» в шапке текущей вкладки → PDF со всеми KPI, графиками, таблицами этой вкладки.
- [ ] Имя файла включает название вкладки и дату.

## Раздел 5 — История отчётов

### 5.1 Список
- [ ] Колонки: Дата, Тип, Описание, SHA256 (короткий), Действия.
- [ ] Сортировка по дате (новые сверху).
- [ ] Фильтр по типу (Аналитика / Form100 / Все).
- [ ] Поиск по описанию.
- [ ] Период «от-до» для фильтрации.

### 5.2 Действия
- [ ] «Открыть» — открывает файл в системном просмотрщике.
- [ ] «Сохранить как» — копирует файл в выбранное место.
- [ ] «Проверить хеш» — пересчитывает SHA256 и сравнивает с записанным.
- [ ] При несовпадении SHA — красный индикатор и предупреждение.
- [ ] При совпадении — зелёный чек.

## Раздел 6 — Drill-down и навигация (новая фича)

- [ ] Клик по KPI «Случаев ИСМП» на Обзоре → переключение на вкладку «ИСМП».
- [ ] Клик по KPI «Положительных» → переключение на вкладку «Микробиология».
- [ ] Клик по строке в Heatmap (Отделение × Микро) → вкладка «Поиск» с применёнными фильтрами.
- [ ] Клик по бару в «Топ микроорганизмов» → вкладка «Микробиология» с выбранным микро.
- [ ] При drill-down период и категория пациента сохраняются.

## Раздел 7 — UX состояния

### 7.1 Empty states
- [ ] За период нет данных → плашка «За выбранный период данных нет. Попробуйте расширить период или сбросить фильтры.»
- [ ] Нет ИСМП за период → плашка «Случаев ИСМП за выбранный период не зарегистрировано» (это позитивное состояние, не ошибка).
- [ ] Нет сохранённых фильтров → плашка «Вы ещё не сохранили фильтры. Настройте поиск и нажмите «Сохранить».»
- [ ] Нет истории отчётов → плашка с подсказкой «Здесь будут отчёты, которые вы сформируете».

### 7.2 Loading states
- [ ] При первой загрузке вкладки — skeleton-плашки на месте KPI и графиков.
- [ ] При обновлении после смены фильтров — лёгкий overlay с прогресс-индикатором, существующие данные не пропадают.

### 7.3 Error states
- [ ] Сетевая ошибка / ошибка БД → inline-плашка в верху страницы, кнопка «Повторить».
- [ ] Ошибка экспорта → inline-уведомление, не модальный диалог.
- [ ] **НЕТ** модальных QMessageBox для нефатальных ошибок.

## Раздел 8 — Адаптивность

- [ ] На ширине экрана 1920+ — KPI-cards в один ряд (4 шт).
- [ ] На ширине 1280–1920 — KPI в один ряд, чарты в одну колонку.
- [ ] На ширине 1024–1280 — KPI в 2 ряда по 2.
- [ ] Tabs остаются всегда видимыми, на узких экранах используется горизонтальный скролл.
- [ ] Sticky filter bar остаётся прилипшим при скролле любой вкладки.

## Раздел 9 — Производительность

## Готовность к выкатке

v2 признаётся готовым к выкатке как default, когда:
- ✅ Все чеклисты в разделах 1–11 выполнены.
- ✅ `pytest` проходит на 100%.
- ✅ Ручной регресс по всем сценариям выполнен и задокументирован.
- ✅ Хотя бы одна неделя в статусе «доступен через флаг» для опционального тестирования.
- ✅ Для каждой новой фичи (heatmap, resistance, drill-down, quick chips) — минимум 3 unit-теста.
- ✅ В `docs/user_guide.md` описаны новые вкладки.
