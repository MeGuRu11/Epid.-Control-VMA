# Спецификация: детализация медицинской помощи в Ф-100

Дата: 2026-04-25
Автор: Codex + пользователь
Статус: реализована

## Описание

В разделе 3 мастера Ф-100 `Мед. помощь` нужно добавить возможность фиксировать текстовые пояснения для всех пунктов медицинской помощи, которые сейчас представлены только чекбоксами. Также нужно добавить отдельный пункт `Оперативное вмешательство`.

Изменение не затрагивает схему БД: карточка Ф-100 уже хранит структурированный JSON payload.

## User Stories

1. Как медицинский сотрудник, я хочу отметить факт выполненной медицинской помощи и сразу вписать дозу, тип, объём или пояснение, чтобы карточка Ф-100 была полной.
2. Как медицинский сотрудник, я хочу отдельно отметить оперативное вмешательство и описать его, чтобы оно попадало в сохранение и экспорт.
3. Как пользователь редактора Ф-100, я хочу видеть те же поля, что и в мастере, чтобы данные не терялись при повторном открытии карточки.

## Модель данных

Новые таблицы и миграции не требуются.

В `data.medical_help` добавляются JSON-ключи:

| Поле | Тип | Описание |
|------|-----|----------|
| `mp_serum_pss_details` | string | Пояснение, доза или серия для ПСС |
| `mp_serum_pgs_details` | string | Пояснение, доза или серия для ПГС |
| `mp_transfusion_blood_details` | string | Что перелито и объём |
| `mp_transfusion_substitute_details` | string | Кровезаменитель и объём |
| `mp_immobilization_details` | string | Вид иммобилизации |
| `mp_bandage_details` | string | Тип перевязки |
| `mp_surgical_intervention` | bool | Выполнено оперативное вмешательство |
| `mp_surgical_intervention_details` | string | Описание оперативного вмешательства |

Старый ключ `mp_serum_dose` сохраняется для обратной совместимости. При загрузке старых карточек он используется как fallback для новых полей сывороток.

## UI

Затрагиваются:

| Виджет | Изменение |
|--------|-----------|
| `StepMedical` | Добавить текстовые поля рядом с ПСС, ПГС, переливанием крови, кровезаменителями, иммобилизацией, перевязкой и новым оперативным вмешательством |
| `Form100EditorV2` | Синхронизировать полный редактор с мастером |
| `Form100MainWidget` | Синхронизировать legacy/встроенный основной виджет с новым набором полей |
| `StepEvacuation` | Показывать оперативное вмешательство в итоговом обзоре |

## Application и отчёты

- `Form100MedicalHelpPayloadInput` расширяется новыми необязательными полями с безопасными значениями по умолчанию.
- PDF-экспорт Ф-100 должен выводить новые текстовые пояснения и строку `Оперативное вмешательство`.
- Валидация существующих обязательных деталей антибиотика и обезболивания сохраняется без расширения на все новые поля.

## Edge Cases

1. Старые карточки содержат только `mp_serum_dose`: значение должно отображаться для сывороток.
2. Новая карточка содержит отдельные детали ПСС/ПГС: значения должны сохраняться отдельно.
3. Чекбокс выключен, но текст введён: текст сохраняется, чтобы пользователь не терял введённые пояснения.
4. Подписанная карточка: новые поля блокируются через существующий `set_locked`/`set_read_only`.

## Тесты

Покрыть:

1. `StepMedical` собирает и загружает новые поля, включая fallback `mp_serum_dose`.
2. `_build_structured_data` и `_build_wizard_payload` сохраняют новый bool/key-набор.
3. `Form100EditorV2` строит payload с новыми полями и загружает их из карточки.
4. PDF-экспорт принимает карточку с новыми полями без ошибки.

## Зависимости

- БД и Alembic не меняются.
- Внешний формат существующих карточек сохраняет обратную совместимость.
- Новые поля добавляются только в существующий JSON payload Ф-100.
