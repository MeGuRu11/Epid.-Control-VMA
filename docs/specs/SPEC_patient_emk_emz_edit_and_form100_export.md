# Спецификация: редактирование ЭМЗ из ЭМК и экспорт отчёта Формы 100

Дата: 2026-04-24
Автор: Codex + пользователь
Статус: утверждена к реализации

## Описание

Кнопка редактирования в разделе `Поиск и ЭМК` должна открывать только редактирование выбранной или последней ЭМЗ. Вкладки `Пациент` и `Форма 100` из этого диалога удаляются, потому что встроенная Форма 100 плохо помещается и ухудшает работу с карточкой.

Отдельно нужно сделать доступным экспорт PDF-отчёта Формы 100 из текущего пользовательского пути: пользователь открывает `Форма 100` из ЭМК, создаёт или редактирует карточку через мастер, затем может выгрузить PDF выбранной карточки из списка Формы 100.

## User Stories

1. Как медицинский сотрудник, я хочу нажать `Редактировать ЭМЗ` в `Поиск и ЭМК`, чтобы сразу попасть в редактирование госпитализации без лишних вкладок.
2. Как медицинский сотрудник, я хочу получить предупреждение, если у выбранного пациента нет госпитализаций для редактирования.
3. Как медицинский сотрудник, я хочу после создания Формы 100 в мастере выбрать карточку и выгрузить PDF-отчёт.
4. Как оператор, я хочу, чтобы экспорт PDF работал без изменения схемы БД и через существующий `Form100ServiceV2.export_pdf(...)`.

Новые application-сервисы не требуются.

- `Form100ServiceV2.export_pdf(...)` для генерации PDF-отчёта выбранной карточки.

## UI (PySide6)

| Виджет | Изменение |
|--------|-----------|
| `PatientEmkView` | Кнопка редактирования переименована в `Редактировать ЭМЗ`; при отсутствии ЭМЗ показывает warning. |
| `PatientFullEditDialog` | Упрощён до одного встроенного `EmzForm` без вкладок пациента и Формы 100. |
| `Form100ListPanel` | В preview-панель добавлена кнопка `Экспорт PDF` для выбранной карточки. |

## Бизнес-логика

1. `PatientEmkView._open_edit_patient()` определяет текущую ЭМЗ; если она не выбрана, берёт последнюю.
2. Если ЭМЗ нет, callback редактирования не вызывается, пользователь видит предупреждение.
3. `MainWindow._open_patient_edit_dialog(patient_id, emr_case_id=None)` открывает упрощённый диалог редактирования ЭМЗ.
4. После закрытия диалога с сохранениями вызывается существующее обновление контекста.
5. `Form100ListPanel._export_selected_pdf()` берёт выбранную карточку, спрашивает путь через `QFileDialog` и вызывает `form100_service.export_pdf(...)`.
6. После успешного экспорта список карточек обновляется, чтобы подтянуть обновлённый artifact metadata.

## Edge Cases

1. Пациент не выбран: остаётся текущий warning.
2. Пациент выбран, но ЭМЗ нет: warning `Нет госпитализаций для редактирования ЭМЗ`.
3. Карточка Формы 100 не выбрана: экспорт PDF ничего не делает.
4. Пользователь отменил выбор пути: экспорт PDF не запускается.
5. Сервис экспорта вернул ошибку: показывается error dialog.

## Безопасность

Новые security-механизмы не вводятся. Экспорт PDF использует существующий сервисный путь и audit-поведение `Form100ServiceV2`. Вопросы шифрования и защиты артефактов остаются вне этой итерации.

## Тесты

1. `PatientEmkView`: редактирование передаёт выбранную/последнюю ЭМЗ и не открывает диалог без ЭМЗ.
2. `PatientFullEditDialog`: содержит только `EmzForm`, включает `edit_mode=True`, загружает выбранную ЭМЗ.
3. `Form100ListPanel`: кнопка `Экспорт PDF` вызывает `form100_service.export_pdf(...)` для выбранной карточки.
4. Regression: эвакуационный блок Формы 100 не меняется.

## Зависимости

- Затронутые слои: UI, tests, docs.
- БД, миграции, domain и infrastructure не меняются.
