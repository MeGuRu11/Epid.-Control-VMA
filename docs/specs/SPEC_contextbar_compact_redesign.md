# Спецификация: компактный редизайн contextbar

Дата: 2026-04-27
Автор: Codex + пользователь
Статус: утверждена к реализации

## Описание

Контекстная панель пациента должна стать компактной однострочной панелью закрепления текущего рабочего контекста. Панель хранит один активный контекст: пациента и, при наличии, госпитализацию. Быстрые переходы между разделами удаляются из панели, потому что основная навигация уже находится в верхнем меню и профильных экранах.

## User Stories

1. Как пользователь, я хочу видеть текущего закреплённого пациента и госпитализацию в одной компактной строке, чтобы панель не занимала значительную часть экрана.
2. Как пользователь, я хочу раскрывать выбор пациента и госпитализации только при необходимости, чтобы обычная работа оставалась плотной.
3. Как пользователь, я хочу очищать пациента или только госпитализацию прямо из чипов, чтобы быстро сменить рабочий контекст.

## UI

| Виджет | Тип | Описание |
|--------|-----|----------|
| `ContextBar` | `QWidget` | Компактная панель закрепления пациента и госпитализации. |

Состояния:

- Свернутое: видна одна строка с заголовком `Контекст пациента`, чипами пациента/госпитализации и действиями `Изменить`, `Последний`, `Сбросить`.
- Раскрытое: под строкой показываются поля `ФИО или ID пациента` и `Номер истории болезни`, кнопки `Найти` и `Выбрать по ID`.
- Нет пациента: чип пациента показывает `Пациент не выбран`, чип госпитализации — `Госпитализация не выбрана`.
- Есть пациент без госпитализации: чип пациента показывает ID и ФИО, чип госпитализации остаётся в незакреплённом состоянии.

## Логика

1. `ContextBar` хранит только текущие `patient_id`, `case_id`, `patient_name`, а также последнего пациента в рамках текущей сессии.
2. Очистка пациента сбрасывает пациента и госпитализацию.
3. Очистка госпитализации оставляет пациента.
4. Кнопка `Последний` восстанавливает последнего выбранного пациента без госпитализации.
5. Контракт `on_quick_action` удаляется; `ContextBar` больше не отвечает за навигацию.

## Безопасность

- Нового пользовательского ввода с записью в БД нет.
- Поиск пациента и госпитализации использует существующие application-сервисы.
- Персональные данные не логируются.
- Аудит не нужен, потому что операции только меняют локальный UI-контекст.

## Тесты

1. `ContextBar`: свернутое состояние по умолчанию, раскрытие/сворачивание по кнопке `Изменить`.
2. `ContextBar`: чипы пустого и выбранного контекста.
3. `ContextBar`: очистка пациента, очистка госпитализации, сброс полей поиска.
4. `ContextBar`: восстановление последнего пациента.
5. `ContextBar`: отсутствие быстрых кнопок разделов и удаление `on_quick_action` из конструктора.
6. `MainWindow`: создание панели без quick-action callback, синхронизация контекста с ЭМЗ, ЭМК и Лабораторией сохраняется.

## Оценка

- Слои: UI и документация.
- БД и миграции: не нужны.
- Сервисные контракты: не меняются.
