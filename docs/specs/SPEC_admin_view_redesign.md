# Спецификация: редизайн страницы «Администрирование»

Дата: 2026-04-25
Автор: Codex + пользователь
Статус: утверждена к реализации

## Описание

Страница `Администрирование` переводится из одной длинной формы в практичную вкладочную рабочую область: `Пользователи`, `Аудит`, `Резервные копии`. Первая итерация фокусируется на управлении пользователями в формате master-detail: список и поиск слева, карточка выбранного пользователя и форма создания справа.

БД, миграции, роли, сервисные контракты и бизнес-логика не меняются.

## User Stories

1. Как администратор, я хочу видеть пользователей отдельной вкладкой со списком и карточкой выбранной записи, чтобы быстрее понимать, с кем выполняю действие.
2. Как администратор, я хочу, чтобы кнопки `Сбросить пароль`, `Активировать`, `Деактивировать` включались только при выбранном пользователе, чтобы не запускать операции без контекста.
3. Как администратор, я хочу подтверждать деактивацию пользователя, чтобы случайно не отключить учётную запись.
4. Как администратор, я хочу видеть аудит и резервные копии в отдельных вкладках, чтобы административные задачи не смешивались в одной длинной странице.
5. Как оператор без прав администрирования, я не должен получать доступ к мутирующим действиям страницы.

## UI (PySide6)

### Виджеты

| Виджет | Тип | Описание |
|--------|-----|----------|
| `UserAdminView` | `QWidget` | Основной экран администрирования |
| `admin_tabs` | `QTabWidget` | Вкладки `Пользователи`, `Аудит`, `Резервные копии` |
| `user_table` | `QTableWidget` | Список пользователей с payload в `Qt.UserRole` |
| `audit_table` | `QTableWidget` | Последние события аудита |
| `backup_create_btn` | `QPushButton` | Создание резервной копии |
| `backup_restore_btn` | `QPushButton` | Восстановление из файла |

### Вкладка «Пользователи»

- Слева: поиск по логину и таблица пользователей.
- Справа: карточка выбранного пользователя с ID, логином, ролью и статусом.
- Ниже карточки: действия выбранного пользователя.
- Отдельно ниже: inline-форма создания пользователя.
- Payload пользователя хранится в `Qt.UserRole`, чтобы действия не зависели от текста ячеек.
- При отсутствии выбора действия выбранного пользователя отключены, статус сообщает `Пользователь не выбран`.
- При деактивации показывается `exec_message_box` с кнопками `Yes/No`, default `No`.

### Вкладка «Аудит»

- Показывает текущий список последних событий.
- При пустом списке отображает понятный empty state без ошибки.

### Вкладка «Резервные копии»

- Показывает текущий статус последней резервной копии.
- Сохраняет существующие действия `Создать резервную копию` и `Восстановить из файла`.
- Новые backend-функции, фильтры и изменения backup-логики не добавляются.

## Edge Cases

1. Список пользователей пуст: таблица пустая, detail-панель показывает `Пользователь не выбран`, действия отключены.
2. Поиск ничего не нашёл: показывается empty state, действия отключены.
3. Нет выбранного пользователя: `_reset_password()` и `_set_active()` не вызывают сервис.
4. Пользователь отказался от деактивации: `set_active(..., False, ...)` не вызывается, таблица не перезагружается.
5. Аудит пустой: вкладка показывает empty state.
6. Backup-сервис занят: backup-кнопки временно отключаются по существующей busy-политике.
7. Роль без прав: мутирующие controls отключены.

## Безопасность

- Проверки прав остаются через текущие `can_manage_users` и `can_manage_backups`.
- Новые права и роли не добавляются.
- Шифрование backup/export и защита артефактов не входят в эту итерацию.
