# Сборка и выпуск Epid Control

## 1. Назначение документа

Документ описывает, как подготовить релиз, собрать `EXE`, собрать установщики и провести минимальную приёмку артефактов.

## 2. Что изменилось в сценариях сборки

Актуальные сборочные сценарии стали более информативными:

- PowerShell-сценарии показывают пошаговый прогресс по этапам сборки.
- В консоли выводятся понятные статусы `INFO`, `OK`, `WARN`, `ERROR`, `DONE`.
- В конце каждого сценария выводится явное финальное сообщение: `Всё готово`.
- Для `NSIS` и `Inno Setup` обновлены приветственные и финальные тексты мастеров.
- Пользователь при установке видит стандартный progress bar мастера и понятное завершение установки.

Общий helper для консольного UX: `scripts/build_ui.ps1`.

## 3. Предварительные условия

Перед сборкой убедитесь, что:

- установлен `Python 3.11+` (`3.12` рекомендуется);
- создано и активировано виртуальное окружение;
- установлены зависимости из `requirements-dev.txt`;
- доступен `PyInstaller`;
- для сборки установщиков установлен хотя бы один упаковщик:
  - `NSIS` для `scripts\build_nsis.bat` и `scripts\build_installer_nsis.ps1`;
  - `Inno Setup 6` для `scripts\build_installer.ps1`.

Подготовка окружения:

```powershell
python -m venv venv
venv\Scripts\activate
pip install -r requirements-dev.txt
```

## 4. Обязательный quality gate перед релизом

Перед любой сборкой выполните полный локальный прогон:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\quality_gates.ps1
```

Релиз собирается только если этот прогон полностью зелёный.

Скрипт проверяет:

1. `ruff check app tests`
2. `python scripts/check_architecture.py`
3. `mypy app tests`
4. `pytest -q`
5. `python -m compileall -q app tests scripts`
6. `python -m alembic upgrade head`
7. `python -m alembic check`
8. `python scripts/check_mojibake.py`

## 5. Сборка исполняемого файла

Основная команда:

```powershell
scripts\build_exe.bat
```

Внутри вызывается `scripts\build_windows.ps1`.

### Что делает сборка

1. Проверяет структуру проекта и наличие `EpidControl.spec`.
2. Находит рабочий интерпретатор Python.
3. Проверяет наличие `PyInstaller`.
4. При обычном запуске очищает `build/` и `dist/`.
5. Собирает `EpidControl.exe` через `PyInstaller`.
6. Создаёт `dist\RELEASE_INFO.txt`.
7. Запускает `scripts\verify_exe.ps1`.

### Что увидит пользователь в консоли

Сценарий показывает:

- заголовок пакета сборки;
- прогресс по шагам;
- диагностические сообщения по найденным путям и инструментам;
- финальный список артефактов;
- следующий шаг: сборка `NSIS` или `Inno Setup`.

### Параметры `build_windows.ps1`

- `-SkipClean` — не очищать `build/` и `dist/`;
- `-SkipVerify` — пропустить проверку `EXE`.

Пример:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_windows.ps1 -SkipVerify
```

### Ожидаемые артефакты

- `dist\EpidControl.exe`
- `dist\RELEASE_INFO.txt`

## 6. Проверка собранного EXE

Скрипт проверки:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\verify_exe.ps1
```

Проверяется:

- существует ли `dist\EpidControl.exe`;
- найден ли runtime в `_internal` для собранного дистрибутива;
- создан ли `dist\RELEASE_INFO.txt`.

Если проверка упала, установщики собирать не нужно — сначала исправьте проблему в `EXE`.

## 7. Сборка NSIS-установщика

Команда:

```powershell
scripts\build_nsis.bat
```

Внутри вызывается `scripts\build_installer_nsis.ps1`.

### Что делает сборка

1. Определяет версию приложения.
2. Проверяет наличие `dist\EpidControl.exe`.
3. Проверяет наличие `scripts\installer.nsi`.
4. Ищет `makensis.exe` в `PATH` или в стандартном каталоге `NSIS`.
5. Собирает `dist\EpidControlSetup_NSIS.exe`.
6. Выводит финальную сводку по артефактам.

### Что изменено в UX NSIS

- приветственная страница стала информативнее;
- пользователь заранее видит, что будут установлены приложение, миграции и деинсталлятор;
- улучшены описания компонент и ярлыков;
- финальная страница явно сообщает: `Всё готово`;
- доступен запуск приложения прямо с финальной страницы мастера.

### Параметры

Поддерживаются:

- `-Version`
- `-Publisher`
- `-AppName`

Пример:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_installer_nsis.ps1 -AppName "Epid Control VMA"
```

### Ожидаемый артефакт

- `dist\EpidControlSetup_NSIS.exe`

## 8. Сборка Inno Setup-установщика

Команда:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_installer.ps1
```

### Что делает сборка

1. Определяет версию приложения.
2. Проверяет наличие `dist\EpidControl.exe`.
3. Проверяет наличие `scripts\installer.iss`.
4. Ищет `ISCC.exe` в `PATH` или в стандартном каталоге `Inno Setup 6`.
5. Собирает `dist\EpidControlSetup.exe`.
6. Выводит финальную сводку по артефактам.

### Что изменено в UX Inno Setup

- приветственная страница теперь объясняет, что именно будет установлено;
- экран готовности установки формулирует понятный статус перед стартом копирования;
- финальная страница показывает явное сообщение `Всё готово`;
- пользователь может сразу запустить приложение после установки;
- описания задач и ярлыков переведены на понятный русский текст.

### Параметры

Поддерживаются:

- `-Version`
- `-Publisher`
- `-AppName`

Пример:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_installer.ps1 -Publisher "MeGuRu11"
```

### Ожидаемый артефакт

- `dist\EpidControlSetup.exe`

## 9. Рекомендуемый релизный сценарий

### 9.1 Подготовка

1. Обновите рабочую ветку.
2. Убедитесь, что документация синхронизирована.
3. Выполните `scripts\quality_gates.ps1`.

### 9.2 Сборка

1. Соберите `EXE` через `scripts\build_exe.bat`.
2. Убедитесь, что `verify_exe.ps1` завершился успешно.
3. Соберите минимум один установщик:
   - `scripts\build_nsis.bat`, или
   - `powershell -ExecutionPolicy Bypass -File scripts\build_installer.ps1`.

### 9.3 Приёмка артефакта

1. Установите приложение на чистую машину или чистый профиль Windows.
2. Проверьте приветственную страницу мастера установки.
3. Проверьте, что прогресс установки отображается штатно.
4. Проверьте финальную страницу и сообщение `Всё готово`.
5. Запустите приложение после установки.
6. Прогоните ключевые сценарии из `docs/manual_regression_scenarios.md`.

## 10. Минимальный smoke после сборки

После сборки проверьте хотя бы следующее:

1. приложение стартует без падения;
2. создаётся или открывается рабочая БД;
3. работает логин;
4. открываются основные разделы;
5. выполняется хотя бы один экспорт отчёта;
6. под ролью `admin` открывается `Администрирование`.

## 11. Частые проблемы

### 11.1 `PyInstaller` не найден

Причина:

- зависимости не установлены в активном интерпретаторе.

Что делать:

- активировать нужное окружение;
- выполнить `pip install -r requirements-dev.txt`.

### 11.2 `makensis.exe` не найден

Причина:

- `NSIS` не установлен или не добавлен в `PATH`.

Что делать:

- установить `NSIS`;
- либо добавить путь к `makensis.exe` в `PATH`.

### 11.3 `ISCC.exe` не найден

Причина:

- `Inno Setup` не установлен или не найден по стандартному пути.

Что делать:

- установить `Inno Setup 6`;
- добавить путь к `ISCC.exe` в `PATH`.

### 11.4 Не найден `dist\EpidControl.exe`

Причина:

- установщик запускается до сборки основного `EXE`.

Что делать:

- сначала выполнить `scripts\build_exe.bat`.

### 11.5 Ошибки доступа при сборке

Причина:

- другой процесс держит `dist\EpidControl.exe` или каталог артефактов.

Что делать:

- закрыть процессы, использующие артефакт;
- при необходимости открыть терминал с повышенными правами;
- повторить сборку.

## 12. Что прикладывать к релизу

Минимальный комплект:

- собранный `EXE` или установщик;
- `RELEASE_INFO.txt`;
- ссылка на коммит или тег релиза;
- отметка о прохождении quality gates;
- отметка о прохождении ручной регрессии.
